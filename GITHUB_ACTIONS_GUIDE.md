# Github Actions 自动签到配置指南

## 功能说明

本功能通过 Github Actions 实现 MineBBS 论坛的自动签到，每天北京时间 08:00 自动执行。

## 配置步骤

### 1. 获取 Cookie 和 CSRF Token

1. 使用浏览器访问 [MineBBS 论坛](https://www.minebbs.com/)
2. 登录你的账号
3. 按 `F12` 打开开发者工具
4. 刷新页面，在 `网络` (Network) 选项卡中找到第一个请求
5. 查看请求头中的 `Cookie` 字段，复制完整内容
6. 在 `元素` (Elements) 选项卡中找到 `<html>` 标签的 `data-csrf` 属性，复制完整值

### 2. 在 Github 仓库中配置 Secrets

1. 打开你的 Github 仓库
2. 进入 `Settings` -> `Secrets and variables` -> `Actions`
3. 点击 `New repository secret` 添加以下三个 Secrets：

#### MINEBBS_COOKIES (必填)
```
你的完整 Cookie 字符串
```
示例：`xf_user_id=12345; xf_session=abcdef123456; ...`

#### MINEBBS_CSRF_TOKEN (必填)
```
你的 CSRF Token
```
示例：`123456789,abcdef1234567890`

#### MINEBBS_ACCOUNT_NAME (可选)
```
账户名称（用于日志显示）
```
示例：`我的主账号`

默认值：`Github Actions 账户`

#### MINEBBS_SKIP_DELAY (可选)
```
是否跳过随机延迟（用于测试）
```
值：`true` 或 `false`

默认值：`false`

**说明**：
- 设置为 `true` 时，签到前不会执行 1-5 分钟的随机延迟
- 建议在测试配置时使用，生产环境请设置为 `false` 或不设置此 Secret

### 3. 启用 Github Actions

1. 确保你的仓库已启用 Github Actions
2. 如果是私有仓库，确保 Actions 权限已开启

### 4. 验证配置

配置完成后，你可以：

1. 进入仓库的 `Actions` 标签页
2. 找到 `MineBBS Daily Signin` 工作流
3. 点击 `Run workflow` 手动触发一次
4. 查看运行日志确认签到是否成功

## 定时任务说明

- **UTC 时间**: 每天 00:00
- **北京时间**: 每天 08:00
- **时区**: UTC+8

如需修改执行时间，编辑 `.github/workflows/signin.yml` 文件中的 cron 表达式。

### Cron 表达式格式

```
┌───────────── 分钟 (0 - 59)
│ ┌───────────── 小时 (0 - 23)
│ │ ┌───────────── 日期 (1 - 31)
│ │ │ ┌───────────── 月份 (1 - 12)
│ │ │ │ ┌───────────── 星期几 (0 - 6, 0 = 星期日)
│ │ │ │ │
* * * * *
```

### 常见时间配置示例

| 北京时间 | UTC 时间 | Cron 表达式 |
|---------|---------|-------------|
| 08:00   | 00:00   | `0 0 * * *` |
| 09:00   | 01:00   | `0 1 * * *` |
| 12:00   | 04:00   | `0 4 * * *` |
| 18:00   | 10:00   | `0 10 * * *` |
| 20:00   | 12:00   | `0 12 * * *` |

## 随机延迟机制

为了降低被风控系统检测的风险，每次签到前会随机延迟 1-5 分钟。

- **最小延迟**: 1 分钟
- **最大延迟**: 5 分钟
- **延迟范围**: [60000ms, 300000ms]

## 重试机制

为应对临时网络问题，脚本内置了自动重试机制。

- **最大重试次数**: 3 次
- **重试间隔**: 指数退避（2 秒、4 秒、8 秒、最多 10 秒）
- **重试场景**:
  - 检查签到状态失败时
  - 执行签到请求失败时

### 重试日志示例

```
[状态检查] 正在检查签到状态...
[状态检查] 检查失败 (尝试 1/3): getaddrinfo EAI_AGAIN www.minebbs.com
[状态检查] 第 1 次重试...
[状态检查] 重试成功
```

### 调试输出示例

脚本会输出详细的页面内容分析，帮助诊断签到状态判断问题：


## 错误处理

### 常见错误及解决方案

#### 1. 缺少环境变量
```
[严重错误] 缺少必要的环境变量：MINEBBS_COOKIES
```
**解决方案**: 检查 Github Secrets 是否正确配置

#### 2. Cookie 过期
```
[状态检查] 检查签到状态失败
```
**解决方案**: 重新获取 Cookie 并更新 Github Secrets

#### 3. CSRF Token 无效
```
[执行签到] CSRF Token 缺失，无法执行签到
```
**解决方案**: 重新获取 CSRF Token 并更新 Github Secrets

#### 4. 网络错误
```
[执行签到] 签到请求失败
```
**解决方案**: 检查网络连接，稍后重试

## 注意事项

1. **安全性**: 
   - Cookie 和 CSRF Token 是敏感信息，请妥善保管
   - Github Secrets 是加密存储的，不会被泄露
   - 不要将包含凭据的代码提交到公共仓库

2. **有效期**:
   - Cookie 和 CSRF Token 可能会过期
   - 建议定期检查签到日志
   - 发现过期及时更新 Secrets

3. **使用限制**:
   - 请遵守 MineBBS 论坛的使用规则
   - 不要频繁修改签到时间
   - 合理使用自动化工具

4. **监控**:
   - 定期检查 Github Actions 运行日志
   - 关注签到成功率
   - 发现异常及时处理

## 手动触发

除了定时任务外，你还可以手动触发签到：

1. 进入仓库的 `Actions` 标签页
2. 选择 `MineBBS Daily Signin` 工作流
3. 点击 `Run workflow` 按钮
4. 选择分支（默认 main/master）
5. 点击 `Run workflow`

## 禁用/启用工作流

### 禁用
```yaml
on:
  schedule:
    - cron: '0 0 * * *'
  workflow_dispatch:

jobs:
  signin:
    runs-on: ubuntu-latest
    if: false  # 添加此行禁用工作流
```

### 启用
删除或注释掉 `if: false` 行即可重新启用。

## 多账户支持

目前 Github Actions 版本仅支持单账户签到。如需多账户签到，请使用本地部署方式。
